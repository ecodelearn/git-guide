Criando uma conta no GitHub

O GitHub é uma plataforma de hospedagem de código-fonte que usa o Git para controle de versão. Siga os passos abaixo para criar sua conta:

  1. Acesse o site do GitHub: https://github.com
  2. Clique no botão "Sign up" (Inscrever-se) no canto superior direito da página.
  3. Preencha o formulário de inscrição:
    • Digite seu endereço de e-mail
    • Crie uma senha
    • Escolha um nome de usuário
  4. Resolva o desafio de verificação para provar que você não é um robô.
  5. Clique em "Create account" (Criar conta).
  6. Verifique seu e-mail e clique no link de verificação enviado pelo GitHub.
  7. Após a verificação, você será redirecionado para configurar seu perfil.

Configurando seu perfil do GitHub

Após criar sua conta, é importante configurar seu perfil:

  1. Adicione uma foto de perfil clicando no ícone de avatar no canto superior direito e selecionando "Your profile" (Seu perfil).
  2. Clique em "Edit profile" (Editar perfil) para adicionar informações como sua bio, localização, e site pessoal.
  3. Considere criar um README.md personalizado para seu perfil, que aparecerá na sua página principal do GitHub.

Conectando o Git local ao GitHub

Para conectar seu Git local ao GitHub, você precisará configurar a autenticação. O método recomendado é usar chaves SSH. Siga as instruções abaixo de acordo com seu sistema operacional:

Windows

  1. Instale o Git Bash (se ainda não tiver): https://git-scm.com/download/win
  2. Abra o Git Bash
  3. Gere uma chave SSH com o comando:
    ssh-keygen -t ed25519 -C "seu.email@example.com"
  4. Pressione Enter para aceitar o local padrão do arquivo
  5. Digite uma senha segura quando solicitado (opcional, mas recomendado)

macOS

  1. Abra o Terminal
  2. Verifique se o OpenSSH está instalado com o comando:
    ssh -V
  3. Se não estiver instalado, você pode instalá-lo com o Homebrew:
    brew install openssh
  4. Gere uma chave SSH com o comando:
    ssh-keygen -t ed25519 -C "seu.email@example.com"
  5. Pressione Enter para aceitar o local padrão do arquivo
  6. Digite uma senha segura quando solicitado (opcional, mas recomendado)

Linux

  1. Abra o Terminal
  2. Na maioria das distribuições Linux, o OpenSSH já vem instalado. Você pode verificar com:
    ssh -V
  3. Se não estiver instalado, você pode instalá-lo com o gerenciador de pacotes da sua distribuição. Por exemplo, no Ubuntu:
    sudo apt-get install openssh-client
  4. Gere uma chave SSH com o comando:
    ssh-keygen -t ed25519 -C "seu.email@example.com"
  5. Pressione Enter para aceitar o local padrão do arquivo
  6. Digite uma senha segura quando solicitado (opcional, mas recomendado)

Adicionando a chave SSH ao GitHub

  1. Copie o conteúdo da sua chave pública:
    • No Windows (Git Bash): cat ~/.ssh/id_ed25519.pub | clip
    • No macOS: pbcopy < ~/.ssh/id_ed25519.pub
    • No Linux: xclip -sel clip < ~/.ssh/id_ed25519.pub (você pode precisar instalar xclip)
  2. No GitHub, vá para Settings > SSH and GPG keys > New SSH key
  3. Cole a chave no campo "Key" e dê um título descritivo
  4. Clique em "Add SSH key"

Testando a conexão

Para testar se a conexão SSH com o GitHub está funcionando, use o comando:

ssh -T git@github.com

Se você ver uma mensagem de boas-vindas com seu nome de usuário do GitHub, a configuração foi bem-sucedida!
